# Claw CLI Tooling

This document specifies the `claw` CLI binary, its commands, configuration system, and the `claw-lsp` language server. This is the primary user-facing document — it defines the complete install-to-run experience.

---

## 0. The Simplest Possible Path (User North Star)

A new user should be able to go from zero to running their first Claw workflow in under 5 minutes:

```bash
# Step 1: Install the Claw compiler (one-time, global)
npm install -g @claw/cli

# Step 2: Install OpenCode — the execution OS (one-time, global)
curl -fsSL https://opencode.ai/install | bash

# Step 3: Create a project
mkdir my-agents && cd my-agents
claw init

# Step 4: Write your pipeline (edit the generated example.claw)

# Step 5: Compile to OpenCode
claw build

# Step 6: Run it
opencode /MyWorkflow "Apple, Microsoft"
```

That's it. No Docker required. No gateway server to run. No API server to manage. OpenCode handles everything.

---

## 1. Command Overview

The `claw` binary is a standalone Rust executable distributed via NPM (`@claw/cli`).

| Command | Purpose |
|---------|---------|
| `claw init` | Scaffold a new Claw project with example `.claw` file |
| `claw build` | Compile `.claw` source to OpenCode config + SDK files |
| `claw dev` | Watch mode: rebuild on file change, open OpenCode |
| `claw test` | Run `.claw` test blocks offline with mock injection |

## 2. `claw init`

**Usage:** `claw init [--path .] [--force]`

**Behavior:**
1. Detect if OpenCode is installed (`opencode --version`). If not found, print an install reminder but do NOT fail.
2. Generate `example.claw` with a working minimal example (one type, one tool, one agent, one workflow).
3. Generate `claw.json` with sensible defaults.
4. Generate `package.json` with `@claw/cli` devDependency and `@modelcontextprotocol/sdk` dependency.
5. Generate `.gitignore` entries for `generated/`.
6. Generate `scripts/` placeholder directory with stub tool implementations.
7. Print a next-steps message.
8. If `--force` is not set and `claw.json` already exists, exit with error.

**Generated `example.claw`:**
```claw
// example.claw — generated by claw init

type SearchResult {
    url: string
    snippet: string
    confidence_score: float
}

tool WebSearch(query: string) -> SearchResult {
    invoke: module("scripts/search").function("run")
}

client MyClaude {
    provider = "anthropic"
    model = "claude-sonnet-4-6"
}

agent Researcher {
    client = MyClaude
    system_prompt = "You are a precise researcher. Always cite sources."
    tools = [WebSearch]
    settings = {
        max_steps: 5,
        temperature: 0.1
    }
}

workflow FindInfo(topic: string) -> SearchResult {
    let result: SearchResult = execute Researcher.run(
        task: "Find the most relevant info about: ${topic}",
        require_type: SearchResult
    )
    return result
}
```

**Generated `claw.json`:**
```json
{
  "build": {
    "source": "example.claw",
    "language": "opencode",
    "output_dir": "generated"
  }
}
```

**Generated `package.json`:**
```json
{
  "name": "my-claw-project",
  "type": "module",
  "scripts": {
    "build": "claw build",
    "dev": "claw dev"
  },
  "devDependencies": {
    "@claw/cli": "latest"
  },
  "dependencies": {
    "@modelcontextprotocol/sdk": "^1.0.0"
  }
}
```

**Console output on success:**
```
✓ Created example.claw
✓ Created claw.json
✓ Created package.json
✓ Created scripts/search.js (stub)
✓ Created .gitignore

Next steps:
  1. Set your API key:   export ANTHROPIC_API_KEY=sk-...
  2. Install deps:       npm install
  3. Compile:            claw build
  4. Run:                opencode /FindInfo "quantum computing"

Tip: Run `claw dev` to watch for changes and auto-rebuild.
```

**OpenCode not found warning:**
```
⚠  OpenCode not found. Install it to run compiled workflows:
   curl -fsSL https://opencode.ai/install | bash
```

## 3. `claw build`

**Usage:** `claw build [source.claw] [--lang opencode|ts|python] [--watch] [--config claw.json]`

**Default language:** `opencode` (set in `claw.json` or via `--lang`).

**Behavior:**
1. Load `claw.json` if no source argument is provided.
2. Read `.claw` source file.
3. Run full `clawc` pipeline: parse → semantic analysis → TypeBox lowering → code generation.
4. Write output files based on `--lang`:

**`--lang opencode` (default) — output files:**
```
opencode.json                      ← OpenCode provider + MCP config
.opencode/
  agents/
    Researcher.md                  ← one per agent block
  commands/
    FindInfo.md                    ← one per workflow block
generated/
  mcp-server.js                    ← MCP server for all tool blocks
  claw-context.md                  ← project context document
  claw/
    index.ts                       ← TypeScript SDK (also emitted)
    document.json                  ← compiled AST (for tooling)
```

**`--lang ts` — output files:**
```
generated/claw/index.ts            ← Zod schemas + typed async functions
generated/claw/document.json       ← compiled AST
```

**`--lang python` — output files:**
```
generated/claw/__init__.py         ← Pydantic models + typed async functions
generated/claw/document.json       ← compiled AST
```

**Watch Mode (`--watch`):**
- Monitor `.claw` source and `claw.json` for changes.
- On change: re-run full build pipeline.
- Print `rebuilt in {N}ms` on success, print error with line/col caret on failure.
- Uses the `notify` crate (cross-platform). Raw `fs.watch()` is prohibited.

**Exit Codes:**

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Parse error (malformed `.claw` syntax) |
| 2 | Semantic error (type mismatch, undefined reference) |
| 3 | Code generation error (template failure) |
| 4 | I/O error (file not found, permission denied) |

**Error Formatting:**
```
error: undefined tool reference 'FakeTool'
 --> example.claw:15:22
  |
15 |     tools = [WebSearch, FakeTool]
  |                         ^^^^^^^^
```

## 4. `claw dev`

**Usage:** `claw dev [--config claw.json]`

**Behavior:**
1. Load `claw.json`.
2. Run initial `claw build` — fail fast if `.claw` has errors.
3. Enter watch mode on the `.claw` source file.
4. On file change: rebuild and print status.
5. On `Ctrl+C` (`SIGTERM`/`SIGINT`): exit cleanly.

**Console Output:**
```
[dev] built in 42ms
[dev] watching example.claw for changes (ctrl+c to stop)
[dev] rebuilt in 38ms
[dev] error in example.claw:22:5 — type mismatch (see above)
```

**Note:** Unlike the old gateway-based `claw dev`, there is no background server to manage. OpenCode is launched separately by the developer. `claw dev` only handles the compile-watch loop.

## 5. `claw test`

**Usage:** `claw test [source.claw]`

**Behavior:**
1. Parse the `.claw` file.
2. Extract all `mock` and `test` blocks.
3. Run the offline test suite using the compiler's built-in AST evaluator (no LLM calls, no OpenCode required).
4. Print pass/fail results per test block.

**Offline test syntax:**
```claw
mock Researcher("Find info about quantum computing") -> {
    "url": "https://example.com",
    "snippet": "Quantum computers use qubits.",
    "confidence_score": 0.95
}

test "FindInfo workflow returns SearchResult" {
    let result = FindInfo("quantum computing")
    assert(result.confidence_score > 0.9)
    assert(result.url != "")
}
```

**Output:**
```
running 1 test in example.claw
  ✓ FindInfo workflow returns SearchResult (2ms)

test result: ok. 1 passed; 0 failed
```

Full test specification: `specs/08-Testing-Spec.md`, `specs/17-Phase6-Test-Runner-And-Mocks.md`.

## 6. Configuration (`claw.json`)

The `claw.json` file lives in the project root. It is minimal by design — most configuration lives in the `.claw` source file itself or in the generated `opencode.json`.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `build.source` | string | `"example.claw"` | Path to `.claw` source file |
| `build.language` | `"opencode"` \| `"ts"` \| `"python"` | `"opencode"` | Primary codegen target |
| `build.output_dir` | string | `"generated"` | Output directory |

**Minimal `claw.json`:**
```json
{
  "build": {
    "source": "example.claw",
    "language": "opencode"
  }
}
```

**`claw.json` is intentionally lean.** Provider credentials, model selection, and agent configuration are all derived from the `.claw` source. The generated `opencode.json` carries the full runtime config. Do not put LLM API keys in `claw.json`.

## 7. Language Server (`claw-lsp`)

The `claw-lsp` binary provides IDE support for `.claw` files via the Language Server Protocol.

**Capabilities:**

| Feature | Implementation |
|---------|---------------|
| Diagnostics | Reuses `clawc` parser + semantic analyzer, reports errors in real time |
| Completion | Suggests keywords and document symbols (agents, types, tools, workflows) |
| Semantic Tokens | Highlights `.claw` keywords for syntax coloring |

**Architecture:**
- Built with `tower-lsp` in Rust.
- On document open/change: re-parse + semantic analysis → publish diagnostics.
- Completion items rebuilt from current AST on every change.
- Shares the same `parser::parse()` and `semantic::analyze()` — no duplication.

**VSCode:** The `vscode-extension/` package bundles `claw-lsp` and activates on `.claw` files automatically.

## 8. Structured Logging

When `--json` flag is set, CLI output uses newline-delimited JSON:

```json
{"level":"info","event":"build_complete","duration_ms":42}
{"level":"error","event":"parse_error","file":"example.claw","line":15,"column":22,"message":"undefined tool reference 'FakeTool'"}
```

Log levels controlled by `CLAW_LOG_LEVEL` env var: `error`, `warn`, `info`, `debug`. Default: `info`.

---

## 9. OpenCode Detection

`claw init` and `claw build` both run a lightweight OpenCode check:

```bash
opencode --version 2>/dev/null
```

- **Found:** proceed silently.
- **Not found:** print install reminder (warning, not error). The compiler still succeeds because OpenCode is only required at runtime, not compile time.

The check uses `$PATH` resolution only. No hardcoded paths.

## 10. Environment Variables

| Variable | Used By | Purpose |
|----------|---------|---------|
| `ANTHROPIC_API_KEY` | OpenCode (runtime) | Anthropic provider auth |
| `OPENAI_API_KEY` | OpenCode (runtime) | OpenAI provider auth |
| `CLAW_LOG_LEVEL` | `claw` CLI | Log verbosity |
| `CLAW_CONFIG` | `claw` CLI | Override config file path |

API keys are NEVER read by `clawc` or stored in `claw.json`. They are read by OpenCode at runtime from the environment, exactly matching OpenCode's own provider configuration model.
